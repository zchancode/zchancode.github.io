1.Ubuntu20.04
2.cuda11.2

4.tensorflow2.8.0
5.pytorch1.10.0
6.gcc<10
7.python3.10
版本对了就不怕检测不到你的GPU，裸机安装无需戴套

cuda11.2
https://developer.nvidia.com/cuda-11.2.0-download-archive?target_os=Linux&target_arch=x86_64&target_distro=Ubuntu&target_version=2004&target_type=runfilelocal
wget https://developer.download.nvidia.com/compute/cuda/11.2.0/local_installers/cuda_11.2.0_460.27.04_linux.run
sudo sh cuda_11.2.0_460.27.04_linux.run

安装的时候你可以根据自己的情况选择要不要安装显卡驱动，其他都勾上

设置环境变量

他这个装到了/usr/local/cuda里要增加环境变量，不然系统读不出来
编辑 nano ~/.bashrc

export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/usr/local/cuda-11.2/lib64
export PATH=$PATH:/usr/local/cuda-11.2/bin
export CUDA_HOME=$CUDA_HOME:/usr/local/cuda-11.2`

生效source ~/.bashrc

验证

nvcc -V

cudnn8.1 for cuda11.2
https://developer.nvidia.com/rdp/cudnn-archive

找三个deb x86_64

#依次安装
sudo dpkg -i libcudnn8_8.1.1.33-1+cuda11.2_amd64.deb
#运行库
sudo dpkg -i libcudnn8-dev_8.1.1.33-1+cuda11.2_amd64.deb
#开发库
sudo dpkg -i libcudnn8-samples_8.1.1.33-1+cuda11.2_amd64.deb
#demo库
验证cudnn

验证cuDNN可用

# 将cuDNN例子复制到可写路径中
cp -r /usr/src/cudnn_samples_v8/ $HOME
# 转到可写路径
cd  ~/cudnn_samples_v8/mnistCUDNN
# 编译文件。
sudo make clean 
sudo make
# 运行样例程序。
sudo ./mnistCUDNN

如果报错说缺什么h文件，说明还要装两个依赖，你搜一下这个报错就可以搜到,如果再说gcc版本有问题你就要切换gcc版本

安装pytroch1.10.0

看准版本，去官网找命令，网上的都会过期

https://pytorch.org/get-started/previous-versions/
pip install torch==1.10.0+cu111 torchvision==0.11.0+cu111 torchaudio==0.10.0 -f https://download.pytorch.org/whl/torch_stable.html

虽然上面提示是对cuda11.1的但是我试了，可以正常用

tensorflow2.8.0
pip install tensorflow==2.8.0

如果不行就是

pip install tensorflow-gpu==2.8.0
测试是否安装成功
import torch   # 能否调用pytorch库

print(torch.cuda.current_device())   # 输出当前设备（我只有一个GPU为0）
print(torch.cuda.device(0))   # <torch.cuda.device object at 0x7fdfb60aa588>
print(torch.cuda.device_count())  # 输出含有的GPU数目
print(torch.cuda.get_device_name(0))  # 输出GPU名称 --比如1080Ti
x = torch.rand(5, 3)
print(x)  # 输出一个5 x 3 的tenor(张量)
import tensorflow as tf
print('GPU',tf.test.is_gpu_available())
a = tf.constant(2.)
b = tf.constant(4.)
print(a * b)
